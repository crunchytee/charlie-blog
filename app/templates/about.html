Project Definition

The Charlie Blog Project was created to be a lightweight, easily deployable, and easily modifyable blog platform. Both Users and Administrators can create and interact with posts, offering a unique platform where fans and creators have equal rights to share.

Project Technologies

Charlie Blog is built on a small set of open source libraries and technologies, focusing on compatability and ease of development. Amongst those technologies are:

  • Flask - Python-based WSGI web application framework
  • Jinja - HTML templating language
  • Postgresql - Open source database
  • SQLAlchemy - Database ORM (object relational mapper)
  • Material Web - Custom web components
  • WTForms - Form validation and rendering library
  • Heroku - Deployment
  • And more! - Have a look at our GitHub for a comprehensive view of all technologies used

Project Methodology

Charlie Blog was constructed following a plan-based waterfall methodology. Each stage of the model was strictly adhered to, allowing for a robust application to be built that also includes documentation. Learn more about the project specifcations Here.
