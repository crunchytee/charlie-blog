Credits | Charlie Blog 🫶

This page discusses the portions of the project that were completed by each member. Charlie Blog was developed as a group project. Learn more about the authors Here.

📏 Specification

Working from an initial requirements sheet, the Software Requirements Specification document was created by Tosh Roberts Brockway. This document outlined each function of the software that was to be created, and the specifications that the function should follow.

🖌️ Design

With a strong understanding of the project requirements, Tosh and Serhii Demchuck built all the design documents for Charlie Blog. Serhii focused on UI (User Interface) and Use Case diagrams to define the look, feel, and interaction of the software. Tosh did the UML Class Diagrams, Algorithm Design, Database Design, and Stack Design.

🖥️ Implementation

After all design was completerd, Charlie Blog was implemented by Tosh Roberts Brockway according to the design diagrams, and specifications. The current release of Charlie Blog is v1.0

🐛 Testing

With a candidate prepared, Charlie Blog was tested by Alexander Bubienko and AJ To for functionality, Validation, Performance, and Acceptance.

💡 Deployment

Charlie blog was deployed by Tosh to Heroku

🚧 Maintenance

Charlie Blog is maintained by Tosh Roberts Brockway. Future releases, if any, will be pushed to this live URL.

⚙️ Project requirements

Presentation organized by AJ To and Serhii Demchuk. Documentation by all team members.
